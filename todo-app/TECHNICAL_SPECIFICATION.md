# Техническое задание: Система управления задачами с фотоотчетами для Telegram

## 1. Общие сведения
**Наименование системы:** TodoPhoto Bot  
**Цель:** Автоматизация постановки и отслеживания задач с возможностью загрузки фотоотчетов для малой компании.  
**Платформа:** Telegram Web App + Bot API  
**Авторизация:** По номеру телефона (автоматически через Telegram)  

---

## 2. Функциональные требования

### 2.1. Ролевая модель
| Роль | Права |
|------|-------|
| **Сотрудник** | - Создание/редактирование своих задач<br>- Загрузка фото в свои задачи<br>- Просмотр задач своего отдела |
| **Руководитель отдела** | - Все права сотрудника<br>- Назначение задач сотрудникам отдела<br>- Просмотр всех задач отдела |
| **Администратор** | - Управление пользователями и отделами<br>- Доступ ко всем задачам и фото<br>- Настройка системы |

### 2.2. Основной функционал
#### Задачи
- Создание задачи с полями:  
  - Название, описание, дедлайн, приоритет (низкий/средний/высокий)  
  - Исполнитель, отдел, статус (новая/в работе/на проверке/завершена)  
- Фильтрация задач по:  
  - Статусу, отделу, исполнителю, дате создания  
- Уведомления о:  
  - Новых задачах, изменении статуса, приближении дедлайна (за 24 часа)  

#### Фотоотчеты
- **Загрузка:**  
  - Неограниченное количество фото на задачу  
  - Поддержка форматов: JPG, PNG, WEBP  
  - Максимальный размер файла: 50 МБ  
  - Асинхронная загрузка с прогресс-баром  
- **Хранение:**  
  - Персональная папка для каждого пользователя: `/uploads/user_{id}/`  
  - Структура: `user_{id}/task_{id}/{timestamp}_{filename}`  
  - Автоматическое сжатие фото >10 МБ (опционально)  
- **Просмотр:**  
  - Галерея фото в задаче с зумом  
  - Предпросмотр в чате Telegram  

### 2.3. Интерфейс (Telegram Web App)
- **Главный экран:** Список задач с фильтрами  
- **Экран задачи:**  
  - Детали задачи  
  - Кнопка "Загрузить фото" (мультивыбор)  
  - Галерея загруженных фото  
- **Профиль:**  
  - Номер телефона (автоматически из Telegram)  
  - Отдел, роль, статистика (выполнено задач/загружено фото)

## 3. Технические требования

### 3.1. Стек технологий
| Компонент | Технологии |
|-----------|------------|
| **Фронтенд** | React + TypeScript, Telegram Web App SDK, Axios |
| **Бэкенд** | Python 3.11+, FastAPI, Aiogram 3.x |
| **База данных** | PostgreSQL 15+ (таблицы: users, departments, tasks, photos) |
| **Хранение фото** | Локальное хранилище (возможность миграции на S3) |
| **Контейнеризация** | Docker + Docker Compose |
| **Кеширование** | Redis (для сессий и очередей загрузки) |

### 3.2. Безопасность
- Валидация данных через Telegram Web App InitData  
- Ограничение доступа к папкам пользователей (проверка прав в API)  
- HTTPS обязательн для продакшена  
- Резервное копирование БД и фото ежедневно  

### 3.3. Производительность
- Поддержка до 100 одновременных пользователей  
- Загрузка фото до 50 МБ без таймаута (асинхронно)  
- Время отклика API < 200 мс (без учета загрузки файлов)  

---

## 4. Структура базы данных
```sql
-- Таблицы
users (id, telegram_id, phone, name, department_id, role)
departments (id, name, manager_id)
tasks (id, title, description, deadline, priority, status, creator_id, assignee_id, department_id)
photos (id, task_id, user_id, file_path, uploaded_at, file_size)
```

## 5. Этапы разработки

### Этап 1: Прототип (1 неделя)
- Базовая структура проекта  
- Авторизация через Telegram  
- CRUD для задач без фото  

### Этап 2: MVP (2 недели)
- Загрузка/просмотр фото  
- Интеграция с Telegram Web App  
- Ролевая модель  

### Этап 3: Расширение (1 неделя)
- Уведомления  
- Фильтры и поиск  
- Админ-панель  

### Этап 4: Деплой (3 дня)
- Docker-конфигурация  
- Настройка сервера (Nginx, SSL)  
- Тестирование нагрузки
