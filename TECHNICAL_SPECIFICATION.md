# Техническое задание: Система управления задачами с фотоотчетами для Telegram

## 1. Общие сведения
**Наименование проекта:** TaskPhotoBot  
**Цель:** Разработка веб-приложения в Telegram для управления задачами и загрузки фотоотчетов сотрудниками различных отделов компании.  
**Платформа:** Telegram Web App + Bot API  
**Авторизация:** По номеру телефона (через Telegram)  

---

## 2. Ролевая модель
| Роль | Права доступа |
|------|---------------|
| **Сотрудник** | - Создание/редактирование своих задач<br>- Загрузка фото в свою папку (неограниченно)<br>- Просмотр своих задач и фото<br>- Отметка выполнения задач |
| **Руководитель отдела** | - Все права сотрудника<br>- Просмотр задач и фото всех сотрудников своего отдела<br>- Назначение задач сотрудникам отдела<br>- Фильтрация по отделу |
| **Администратор** | - Управление пользователями и отделами<br>- Полный доступ ко всем задачам и фото<br>- Настройка системы |

---

## 3. Функциональные требования

### 3.1. Управление задачами
- Создание задачи с полями: название, описание, дедлайн, приоритет, ответственный, отдел
- Статусы задач: `Новая`, `В работе`, `На проверке`, `Выполнена`, `Отклонена`
- Комментарии к задачам
- Уведомления о новых задачах, изменениях статуса и дедлайнах
- Фильтры: по статусу, отделу, дате, приоритету

### 3.2. Работа с фото (обновлено)
- **Неограниченная загрузка фото** каждым пользователем
- **Персональная папка** для каждого пользователя на сервере:  
  `/storage/users/{user_id}/photos/`  
  Пример: `/storage/users/12345/photos/2024-06-15_14-30-00_image.jpg`
- Автоматическое создание папки при первом входе пользователя
- Поддержка форматов: JPG, PNG, WEBP
- Максимальный размер одного фото: 20 МБ
- Генерация миниатюр (150x150) для быстрого просмотра
- Привязка фото к задачам (опционально) или свободная загрузка
- Просмотр фото в галерее с пагинацией
- Удаление фото только владельцем или администратором

### 3.3. Интеграция с Telegram
- Запуск через кнопку в боте (`/start`)
- Авторизация через данные Telegram (номер телефона автоматически)
- Уведомления в чат Telegram о важных событиях
- Возможность быстрой отправки фото прямо из чата бота (с автоматической загрузкой в папку пользователя)

### 3.4. Отчетность
- Экспорт задач и фотоотчетов в PDF/Excel по отделам/периодам
- Статистика: количество задач по статусам, загруженные фото по пользователям

## 4. Технические требования

### 4.1. Стек технологий
| Компонент | Технология |
|-----------|------------|
| **Frontend** | React + TypeScript, Telegram Web App SDK, TailwindCSS |
| **Backend** | Python 3.11+, FastAPI, Aiogram 3.x |
| **База данных** | PostgreSQL 15+ |
| **Хранение фото** | Локальное файловое хранилище на сервере (с возможностью миграции на S3) |
| **Кеширование** | Redis (для сессий и очередей уведомлений) |
| **Контейнеризация** | Docker, Docker Compose |
| **Деплой** | Nginx + Gunicorn/Uvicorn, SSL (Let's Encrypt) |

### 4.2. Структура хранения фото
```
/storage
├── users/
│   ├── {user_id_1}/
│   │   └── photos/
│   │       ├── 2024-06-15_14-30-00_img1.jpg
│   │       ├── 2024-06-15_14-31-00_img2.jpg
│   │       └── thumbnails/
│   │           └── 2024-06-15_14-30-00_img1_thumb.jpg
│   └── {user_id_2}/
│       └── photos/
└── tasks/ (опционально, если фото привязаны к задачам)
    └── {task_id}/
        └── attachments/
```

### 4.3. Безопасность
- Валидация данных от Telegram (проверка подписи)
- Ограничение доступа к папкам пользователей (только владелец и админ)
- Санитизация имен файлов
- Rate limiting на загрузку фото (макс. 10 фото в минуту)
- HTTPS обязательн

### 4.4. Производительность
- Поддержка до 100 одновременных пользователей
- Загрузка фото до 20 МБ без таймаутов (использование chunked upload)
- Асинхронная обработка фото (генерация миниатюр в фоне через Celery)
- Пагинация в галерее фото (по 20 элементов на страницу)

## 5. Нефункциональные требования
- Время отклика интерфейса < 1 сек
- Доступность 99% (кроме плановых работ)
- Резервное копирование БД и фото ежедневно
- Логирование всех действий пользователей (аудит)

---

## 6. Этапы разработки

### Этап 1: Прототип (2 недели)
- Базовая структура проекта (Docker, БД, авторизация)
- Создание/просмотр задач
- Загрузка одного фото в папку пользователя
- Интеграция с Telegram Web App

### Этап 2: MVP (3 недели)
- Полноценная загрузка неограниченного количества фото
- Галерея фото с миниатюрами
- Ролевая модель (сотрудник/руководитель/админ)
- Уведомления в Telegram
- Фильтры и поиск по задачам

### Этап 3: Расширение (2 недели)
- Экспорт отчетов
- Статистика и дашборды
- Оптимизация загрузки больших объемов фото
- Тестирование нагрузки

### Этап 4: Деплой и документация (1 неделя)
- Настройка продакшн-сервера
- Инструкция для администраторов
- Обучение пользователей
